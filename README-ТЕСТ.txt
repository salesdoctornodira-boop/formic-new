Это полностью рабочая копия приложения с ГОТОВОЙ базой данных (демо-
данные уже залиты — ~1800 оценок, 85 сотрудников). Можно нажимать что
угодно и ломать — это тест, на боевую систему это никак не влияет.

────────────────────────────────────────────────────────────────────
КАК ЗАПУСТИТЬ
────────────────────────────────────────────────────────────────────
• Mac:     дважды кликните  start.command
           (если Mac ругается «неизвестный разработчик» — правый клик →
            «Открыть» → «Открыть»). Ничего устанавливать не нужно.

• Windows: дважды кликните  start.bat
           (PHP уже встроен в сборку — ничего скачивать не нужно).

После запуска само откроется окно браузера:  http://localhost:8080
Чтобы остановить — просто закройте чёрное окно/терминал.

────────────────────────────────────────────────────────────────────
ЛОГИНЫ
────────────────────────────────────────────────────────────────────
• Сотрудник (аноним, по коду):
      на входе вкладка «Сотрудник» → «Получить мой код» →
      выбрать проект + отдел → получить код → «Войти с этим кодом».
      ВАЖНО: код анонимный, привязан к ОТДЕЛУ, не к человеку.

• Руководитель:   вкладка «Руководитель» → выбрать себя → пароль выдаёт администратор

• Админ:          вкладка «Админ» →  логин и пароль выдаёт администратор
• Супер-админ:    вкладка «Админ» →  логин и пароль выдаёт администратор

  ⚠️  В целях безопасности логины и пароли больше НЕ публикуются в этом файле.
      Получите их лично у администратора системы. Не пересылайте их в чатах
      и не записывайте в общедоступных местах.

────────────────────────────────────────────────────────────────────
ЧТО ПРОВЕРИТЬ (сценарии)
────────────────────────────────────────────────────────────────────
1) СОТРУДНИК / опрос (ДВА ШАГА):
   - получить код, оценить свою команду (1–10 по 4 ценностям);
   - ШАГ 1 (обязательный): своя команда (Required) + Highly Recommended +
     свой руководитель и профильный директор (категории Linear Managers /
     Executive Management). Кнопка «Далее» активна, только когда заполнены
     ВСЕ позиции шага 1 (на шаге 1 «Завершить» нет — чтобы не закрыть рано);
   - ШАГ 2 (Recommended): остальные руководители и команды — по желанию.
     Внизу крупная зелёная кнопка «ЗАВЕРШИТЬ»;
   - поставить 8/9/10 — попросит описать КЕЙС (конкретный случай + пример);
   - поставить 1–4 — попросит описать нарушение (красный флаг);
   - в правом верхнем углу — КРАСНЫЙ ТАЙМЕР кода: код живёт 5 ЧАСОВ
     (отсчёт с момента ВЫДАЧИ кода);
   - «один код — одно устройство»: вход по тому же коду с нового устройства
     закрывает старую сессию;
   - «ЗАВЕРШИТЬ» отправляет всё и ОКОНЧАТЕЛЬНО закрывает код (повторно войти
     и править уже нельзя; статус в админке → «Завершил»);
   - попробовать ещё раз «Получить код» с того же устройства → НЕЛЬЗЯ
     («у вас уже есть код»). Коды можно генерировать, пока отдел не закрыт
     (слот занимает только ЗАВЕРШИВШИЙ опрос, не черновик).

2) РУКОВОДИТЕЛЬ:
   - войти по паролю руководителя (выдаёт администратор), поставить оценки (это «Final» — авторитетные);
   - кнопка «Мониторинг» — прогресс прохождения опроса ТОЛЬКО по своим
     отделам/командам (без имён — только счётчики).

3) АДМИН:
   - дашборд аналитики (рейтинг компании, Top-5, отделы, проекты, пончики);
   - в рейтинге отделов — СЕЛЕКТОР МЕТРИКИ: Балл / Голоса / Сумма / Охват%
     (Охват% = сколько из обязанных оценить — реально оценили);
   - вкладка «Модерация (9–10)» — одобрить/отклонить высокие оценки
     (в демо очередь уже наполнена — есть что одобрять/отклонять);
   - вкладка «Red Flags» — низкие оценки с описанием нарушения;
   - ЭКСПОРТ: CSV и .xlsx (Excel/Google Sheets) — имён авторов там НЕТ (анонимно).

4) СУПЕР-АДМИН (всё, что у админа, плюс):
   - вкладка «Прогресс» — анонимный мониторинг прохождения по ВСЕМ отделам
     (получили код / завершили / в процессе / не начинали — только числа);
   - «Настройки» → «Права голосования»: тумблер «Сотрудники оценивают
     руководителей» и переключатель «Каких показывать»: «Только линейный
     менеджер» (по умолчанию — обязателен голова отдела + профильный директор,
     остальные в Recommended) или «Все руководители»;
   - «Настройки» → «Функции платформы»: тумблеры всех новых фич;
     остальное — периоды, лимиты отделов и кодов, веса, пороги,
     пароли, переименование ценностей;
   - «Сотрудники»: добавить / деактивировать / изменить (конструктор;
     для руководителя — подсказка Executive/Linear и его зона оценивания);
   - «Коды»: прогресс по отделам, лимит кодов на отдел, «Активные устройства»
     (грубый тип устройства, без имён/IP), ВОССТАНОВЛЕНИЕ/УДАЛЕНИЕ анкеты
     (если сотрудник потерял код: называет, кого оценил → находим его
      анонимную анкету → удаляем с данными → он берёт новый код);
   - «Журнал раскрытий»: любое служебное раскрытие автора логируется.

5) АНОНИМНОСТЬ (главное):
   - даже супер-админ в обычном виде НЕ видит, кто как голосовал;
   - раскрыть автора можно только намеренно и только супер-админом,
     и это пишется в журнал;
   - в экспорте (CSV/.xlsx) без «деанона» имён авторов нет;
   - мониторинг прогресса и «Активные устройства» — только счётчики,
     без имён и без привязки к конкретному голосу.

6) ЯЗЫК: переключатель RU/UZ в шапке (выбор запоминается).

СБРОС ДАННЫХ (если «наломали» и хотите начать заново)
────────────────────────────────────────────────────────────────────
Замените файл  data/ccs.db  на копию  data/ccs.db.fresh
(скопировать с заменой). Перезапустите start.command / start.bat.
В «fresh» — те же люди и настройки, но БЕЗ оценок (чистый замер).
